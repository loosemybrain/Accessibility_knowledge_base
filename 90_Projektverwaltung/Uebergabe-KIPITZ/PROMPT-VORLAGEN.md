# Prompt-Vorlagen für die Nutzung mit KIPITZ

## Ziel

Diese Datei enthält konkrete Prompt-Vorlagen für die tägliche Arbeit mit KIPITZ. Sie sollen den Nutzer dabei unterstützen, neue Dokumentationen strukturiert, sauber und wiederholbar zu erstellen.

## 1 Prompt für die Projektinitialisierung

```text
Erstelle aus der vorhandenen Vorlage ein neues Dokumentationsprojekt für das Thema [THEMA].

Bitte:
- definiere Ziel, Scope und Zielgruppe,
- ergänze eine saubere Dokumentationsstruktur,
- formuliere eine erste Gliederung,
- identifiziere offene Punkte und notwendige Quellen,
- halte den Stil neutral, klar und verständlich.
```

## 2 Prompt für die Gliederung einer Dokumentation

```text
Erstelle eine strukturierte Gliederung für eine Dokumentation zum Thema [THEMA].

Bitte:
- teile das Thema in sinnvolle Abschnitte ein,
- stelle Einleitung, Hintergrund, Vorgehen, Beispiele und offene Punkte dar,
- formuliere klare Überschriften,
- vermeide unnötige Wiederholungen,
- halte die Struktur so, dass sie direkt in eine Markdown-Datei übernommen werden kann.
```

## 3 Prompt für die Erstellung eines Task-Formats

```text
Erstelle einen Task für die Bearbeitung von [THEMA] im Standardformat des Vorlageprojekts.

Bitte:
- formuliere Ziel und Scope präzise,
- benenne betroffene Dateien,
- beschreibe die Aktion und die Einfügeposition,
- ergänze erwartetes Ergebnis und Validierung,
- vermeide fachliche Annahmen ohne Quelle.
```

## 4 Prompt für die Prüfung eines Entwurfs

```text
Prüfe den vorhandenen Entwurf zur Dokumentation [THEMA].

Bitte:
- prüfe Ziel, Scope und Struktur,
- identifiziere fehlende Abschnitte oder offene Punkte,
- prüfe, ob Quellen oder Evidenzpunkte benannt sind,
- formuliere konkrete Verbesserungsvorschläge,
- halte die Ausgabe knapp und handlungsorientiert.
```

## 5 Prompt für die Freigabevorbereitung

```text
Prüfe, ob die Dokumentation [THEMA] freigabefähig ist.

Bitte:
- prüfe Vollständigkeit und Verständlichkeit,
- prüfe, ob Quellen oder Nachweise benannt sind,
- identifiziere offene Punkte, die noch vor einer Freigabe geklärt werden müssen,
- formuliere die nächsten Schritte möglichst konkret.
```

## 6 Prompt für die Zusammenfassung eines Projektstands

```text
Erstelle eine knappe Zusammenfassung des aktuellen Projektstands zu [THEMA].

Bitte:
- fasst die wichtigsten Fortschritte zusammen,
- listet offene Punkte und Risiken auf,
- benennt den nächsten Schritt,
- halte die Ausgabe so, dass sie direkt in ein Statusdokument übernommen werden kann.
```

## Standardregel für alle Prompts

Jeder Prompt sollte immer mindestens diese vier Anforderungen enthalten:

1. Ziel des Auftrags klar beschreiben.
2. Scope und Umfang festlegen.
3. gewünschtes Ausgabeformat benennen.
4. Qualitäts- und Prüfregeln ausdrücklich mitgeben.
